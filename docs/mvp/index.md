# MVP 版本概述

## 目标与范围

Delphi MVP 的目标是交付一个**可本地部署、可实际使用**的 RAG 知识库系统最小闭环。用户能够将代码仓库或文档目录导入系统，并通过自然语言提问获得基于知识库内容的回答。

MVP 不追求功能完整性，而是验证核心链路的可行性：**导入 → 索引 → 检索 → 生成**。

---

## 核心用户故事

1. **作为开发者**，我希望将一个 Git 仓库导入 Delphi，以便向代码库提问（如"这个函数的作用是什么"）。
2. **作为文档维护者**，我希望将一个本地文档目录导入 Delphi，以便快速检索文档内容。
3. **作为用户**，我希望用自然语言提问，系统返回带有来源引用的回答，而不是裸文本。
4. **作为运维人员**，我希望通过 Docker Compose 一键启动整个系统，无需手动配置依赖。

---

## 功能边界

### MVP 包含

- Git 仓库导入（远程 URL 或本地路径）
- 文档目录导入（Markdown、HTML、PDF、TXT）
- 代码文件解析与 Tree-sitter 语法感知切分
- 文档按标题层级切分
- Embedding 生成（BGE-M3 本地模型）
- 向量存储（Qdrant）
- 基础 RAG Pipeline：检索 → Prompt 组装 → LLM 生成
- 流式输出（SSE）
- 回答中附带来源引用（文件路径 + 行号或标题）
- REST API（FastAPI）
- Docker Compose 一键部署

### MVP 不包含

- Web UI（仅 API）
- 多用户 / 权限管理
- 增量更新（仓库变更后重新全量导入）
- 混合检索（BM25 + 向量）
- Reranker
- 多模态（图片、音频）
- 自定义 Embedding 模型切换（UI 层）
- 对话历史 / 多轮上下文管理

## 验收标准

### 导入功能

- [ ] 给定一个公开 GitHub 仓库 URL，系统能在 5 分钟内完成 clone、解析、索引（仓库规模 ≤ 50MB）
- [ ] 给定一个本地文档目录（含 Markdown/PDF），系统能正确解析并存入向量数据库
- [ ] 导入完成后，API 返回导入的 Chunk 数量和耗时统计

### 检索与生成

- [ ] 针对已导入内容提问，系统能在 10 秒内返回第一个 SSE token
- [ ] 回答中包含至少一条来源引用，引用指向实际存在的文件
- [ ] 当检索结果为空时，系统返回明确的"未找到相关内容"提示，而非幻觉回答

### 部署

- [ ] `docker compose up` 后，所有服务健康检查通过
- [ ] 提供 `/health` 端点，返回各依赖服务状态

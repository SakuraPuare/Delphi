显存分配（96GB 总显存）：
- Qwen2.5-72B AWQ INT4（~45GB）+ Embedding (1.5GB) + Reranker (1.5GB) + Whisper (3GB) = ~51GB，剩余 45GB 给 KV cache
- 或 Qwen3.5-27B FP16（~54GB）全精度运行，无需量化损失

性能实测数据（来源：CloudRift Benchmark）：

| 模型 | 4× RTX 4090 | 1× RTX PRO 6000 | 对比 |
|------|------------|-----------------|------|
| Qwen3-Coder-30B AWQ | 8,903 tok/s | 8,425 tok/s | 4090 胜 6% |
| Llama-3.3-70B AWQ INT4 | ~900 tok/s（PP=4） | 1,031 tok/s | PRO 6000 胜 15% |
| GLM-4.5-Air AWQ 4bit (~96GB) | 1,731 tok/s | 3,169 tok/s | PRO 6000 胜 83% |

关键注意事项：

1. **无 NVLink**：RTX 4090 不支持 NVLink，多卡通信只能走 PCIe（~32 GB/s），跑 70B+ 模型时跨卡通信是瓶颈
2. **推荐 Pipeline Parallelism**：vLLM 使用 `--pipeline-parallel-size 4` 而非 `--tensor-parallel-size 4`，PP 只在阶段边界传输激活值，对 PCIe 带宽更友好
3. **NUMA 拓扑**：务必确保 4 张卡在同一 NUMA 节点上，否则性能可能下降 35%（参考 vLLM Issue #1838）
4. **功耗散热**：4 卡满载 1800W + CPU/主板约 2200W 总功耗，需要双电源 + 良好散热环境（建议机房部署）
5. **30B 以下模型更划算**：如果主力模型是 27B-30B 级别，4× 4090 可以跑 4 个独立副本做数据并行，吞吐量远超单卡方案

最佳使用策略：
- 27B 及以下模型：每卡跑一个独立副本（DP=4），4 倍吞吐，延迟不变 — **这是 4× 4090 的最大优势**
- 70B 模型：PP=4 或 PP=2+TP=2，可用但延迟比单张大显存卡高
- 不建议跑 100B+ 模型，PCIe 带宽成为严重瓶颈

## 5. 为什么不需要集群

- 200 万 Chunk 的向量索引约 14GB，单机内存放得下
- 27B 量化模型约 20GB 显存，单卡 RTX 5090 够用
- 单机 QPS 足以支撑小团队（10~50 人）并发
- 多机集群引入的运维复杂度远超收益

## 7. 采购建议

1. **首选方案 A（RTX 5090 单卡，~2.9 万）**：最佳平衡点，所有服务全 GPU 加速，Qwen3 全家桶在代码检索上比现有 BGE 方案强一个档次
2. RTX 5090D V2 受出口管制影响可能停售，建议尽早采购；如果买不到，RTX 4090 二手 + Qwen3.5-9B 也完全可用
3. 内存建议 64GB 起步，如果要索引超大代码仓库（200 万+ Chunk）建议 128GB
4. NVMe 建议 2TB，模型权重 + 向量索引 + 代码缓存实际占用约 120GB，但需要留足余量给增量更新和临时文件
5. 电源务必留足余量，RTX 5090 峰值功耗可达 600W+，加上 CPU 和其他组件，1000W 是安全线

# 硬件要求

Delphi 仅需推理能力，训练/微调在外部完成后导入权重即可。

## 推荐配置

| 组件 | 最低要求 | 推荐配置 | 说明 |
|------|---------|---------|------|
| GPU | 1× 16GB 显存 | 1× RTX 4090 (24GB) | LLM 推理 + Embedding |
| CPU | 8 核 | 16+ 核 (Ryzen 9 / i9 / Xeon) | AST 解析、数据预处理 |
| 内存 | 32GB | 64GB~128GB DDR5 | 向量索引 + 模型加载缓冲 |
| 磁盘 | 500GB SSD | 2TB NVMe | 模型权重 20~40GB，向量索引 50~100GB |

## 不同显存下的模型选择

| 显存 | 可用模型 | 量化方式 |
|------|---------|---------|
| 8GB | Qwen2.5-Coder-7B | Q4 |
| 16GB | Qwen2.5-Coder-14B 或 DeepSeek-Coder-V2-Lite | Q4 |
| 24GB | Qwen2.5-Coder-32B | AWQ/Q4 |
| 48GB+ | Qwen2.5-Coder-32B（FP16）| 无需量化 |

## 为什么不需要集群

- 200 万 Chunk 的向量索引约 14GB，单机内存放得下
- 32B 量化模型约 20GB 显存，单卡 4090 够用
- 单机 QPS 足以支撑小团队（10~50 人）并发
- 多机集群引入的运维复杂度远超收益

## 微调环境（独立于部署）

微调不在部署机上进行，可选方案：

| 方案 | 适用场景 |
|------|---------|
| 云端 GPU 实例（A100/H100）按需租用 | 最灵活，按小时计费 |
| 本地多卡工作站 | 频繁微调时更经济 |
| Colab / Kaggle 免费 GPU | 小规模实验 |

微调产物（LoRA 权重或合并后的全量权重）导入部署机即可。
